    Обработчик-генератор адресов

    URL обработчика-генератора адресов используется для создания определенных адресов. В частности - для создания виртуальных адресов.
    Примеры ключей и адресов:
    address #1: 1LotZEBzr53WbXiuuCJagxceP3zvh3sS87
    publicKey #1: 021c9a329a9e8f9ee36540ecdce541f85fdf05215d0634928a52ba1577b801d68e
    privateKey #1: 48e6222b8ea1122557da2dc6ee2f97ca9f63d47013ab024f75e8b2caf79363b2

    address #2: 0xdb0dc469696762a517bcaf451b5e3a1cd0a62d5b
    publicKey #2: 038a74cc5498f1f6b39bd0b38603172d0fe12467fb131eaaa335785e6a36ca9789
    privateKey #2: cda83775ba48e7721c44d1d1418fc11cc15807b2036cb946f1d9b4e4415f13e0

    Входящие данные:
    cryptocurrencyapi.net - номер версии (больше или равен 3)
    chain
    currency
    type - Give
    label
    uniqID
    sign - подпись (хеш данных) для проверки подлинности (см. Алгоритм проверки данных IPN)

    Пример входящих данных:
    [cryptocurrencyapi.net] = 3
    [chain] = ethereum
    [currency] = ETH
    [type] = give
    [label] = My label1
    [sign] = 4a2378f4270c5335f296c9965b3b9601dd42fd09

    Ответ: (в формате JSON)
    publicKey или privateKey

    Пример ответа:
    "038a74cc5498f1f6b39bd0b38603172d0fe12467fb131eaaa335785e6a36ca9789"
